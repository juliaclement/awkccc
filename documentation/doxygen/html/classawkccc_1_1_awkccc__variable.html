◆ compare()

int awkccc::Awkccc_variable::compare ( const Awkccc_variable &  rhs) const
inline

Comparison operators. All comparisons are routed through the single compare() routine which returns negative, 0 or positive with the same meaning as strcmp.

POSIX 2017 states: " Comparisons (with the '<', "<=", "!=", "==", '>', and ">=" operators) shall be made numerically if both operands are numeric, if one is numeric and the other has a string value that is a numeric string, or if one is numeric and the other has the uninitialized value. Otherwise, operands shall be converted to strings as required and a string comparison shall be made as follows: For the "!=" and "==" operators, the strings should be compared to check if they are identical but may be compared using the locale-specific collation sequence to check if they collate equally. For the other operators, the strings shall be compared using the locale-specific collation sequence. The value of the comparison expression shall be 1 if the relation is true, or 0 if the relation is false." FIXME: Haven't yet implemented locale-specific collation as required by the standard

Modulo operator %. C++ standard requires that both operators are an integral type, but we store numbers as double so we force the issue. Datatype long long was chosen as it gives the greatest chance of a correct translation.
